# notion-sync (빌드 참조)

> Notion 읽기/쓰기 코드(`apps/server/src/notion/*`)를 작성/수정할 때 읽는다.

## 절대 규칙 (도메인 불변식 §5)
- **시스템이 쓰는 필드는 `종목 DB.현재가` 1개**(+선택 `데이터 플래그`). 그 외 필드는 **읽기만**.
- **평단가·평가금액·평가손익·수익률·보유수량은 Notion 수식/롤업 소유** — 절대 역방향으로 쓰지 않는다.
- `매매일지.실현손익` 수식은 과대계상 가능성이 있어도(§5.6) **건드리지 않는다.**

## 현재가 upsert (S4 필수)
- **티커/코드를 매칭 키**로 page를 찾는다. 매핑(티커→page_id)은 SQLite에 캐시.
- **~3 req/s 준수**(C5). 시세 틱을 그대로 쓰지 말고 **주기 배치**(1~5분).
- 쓰기 후 검증: 200 + 사용자/수식 소유 필드 미변경.
- 레코드 단위 재시도 → 실패분은 `/output/sync_failures.json` 큐, 다음 사이클 재시도.
- 보유수량/현재가 급변 시 사람 검토 플래그.

## 체결 임포트 (S4 선택, `ENABLE_TRADE_IMPORT`)
- `매매일지`에 **증권사=토스증권 행만** 생성.
- **중복키 = 체결ID/시각 + 종목 + 수량 + 단가.** 중복 0 보장.
- 임포트 커서(마지막 체결 시각/ID)는 `/output/import_cursor.json`에 보관 → 커서 이후만 처리.
- 자동 생성행도 **사용자 편집은 보존**(덮어쓰기 금지).

## 읽기
- 종목 DB → `NotionStockRowSchema`(pageId·ticker·평단가·수량). 위젯 라이브 오버레이·평단가 대조용.
- ETF 구성종목·비중은 사용자가 Notion에 수동 등록(§5.4) → `EtfConstituentSchema`.

## 필드 소유권 표 (부록 B)
- 종목 DB: `현재가`=시스템 쓰기 / 평가·수익률=수식 / 평단가·수량·총보유금액=롤업(대조만) / 종목명·티커·메모=사용자.
- 매매일지: 행=하이브리드(토스 자동+수동) / 회고=사용자.
- 증권사 계좌: 전부 사용자·롤업 — 시스템 무관.
